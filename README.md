# Industry AI Daily Kit

一个面向具体行业的 AI 情报日报 starter kit。

它适合用来搭建“每天自动或半自动收集 AI 与行业动态，再整理成可读日报”的工作流。默认保留大 AI 新闻底盘，同时把重点收窄到某个行业、岗位、产品线或客户场景。

## 它解决什么问题

很多 AI 日报只是在汇总新闻，问题是信息太泛，很难直接服务一个具体行业。

这个项目提供的是一套可迁移的日报骨架：

- 用 AIHOT 做全局 baseline scan
- 用 X / Twitter 捕捉海外一手发布和即时反应
- 用 Reddit 捕捉开发者、用户和社区反馈
- 用知乎补充中文长文和本土行业观点
- 用官方公告、政策网站和公司博客校验事实
- 用通用 WebSearch 补齐行业媒体、报告和长尾来源
- 用“跨平台观察”和“昨日漏项回扫”降低漏报和误判

## 核心原则

1. AIHOT 是基线扫描，不是最终证据源。
2. 重要条目必须尽量追溯到原始来源。
3. 大 AI 新闻要保留，但日报重点应服务具体行业。
4. 同一事件要区分“事实发布”“社区反应”“行业影响”。
5. 先跑通人工半自动流程，再考虑自动化。

## 快速开始

最简单的用法：

1. 打开 [`02_Codex启动Prompt.md`](02_Codex启动Prompt.md)
2. 把全文复制到 Codex
3. 按 Codex 的问题回答你的行业、关注方向、信息源和输出要求
4. 生成第一版行业日报配置
5. 连续试跑 3 天，再决定是否写脚本、定时任务或推送流程

## 推荐日报结构

```md
# 行业 AI 情报日报

## 1. 今日最重要的 3 条
## 2. 大 AI 新闻底盘
## 3. 本行业 AI 动态
## 4. 竞品和公司动作
## 5. 社区反馈
## 6. 政策、监管和官方公告
## 7. 跨平台观察
## 8. 昨日漏项回扫
## 9. 明日跟进清单
```

## 适合扩展的方向

- 增加 RSS、GitHub、Product Hunt、Hacker News 等来源
- 接入定时任务
- 生成 Markdown 存档
- 推送到飞书、邮件或团队知识库
- 生成周报、月报和长期趋势库
- 沉淀成竞品库、机会库和内容选题库
