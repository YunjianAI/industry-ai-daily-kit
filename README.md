# Industry AI Daily Kit

这是一个可移植的行业 AI 情报日报 starter kit。

它不是只调用 AIHOT 的单点工具，而是一套多源信息收集、交叉验证、行业化筛选和日报输出流程。适合把通用 AI 新闻底盘，改造成某个具体行业的每日情报简报。

## 适合谁

- 需要每天跟踪 AI 大新闻的人
- 需要关注 AI 对某个行业影响的人
- 需要把 X、Reddit、知乎、官网、政策网站、行业媒体的信息整理成日报的人
- 想先用 Codex 跑通人工半自动流程，再逐步自动化的人

## 核心原则

1. AIHOT 只作为 baseline scan，不作为最终证据源。
2. 大 AI 新闻保留，但日报重点要偏向使用者所在行业。
3. X、Reddit、知乎、官方公告、行业媒体和 WebSearch 需要交叉验证。
4. 每条重要信息要能追溯到原始来源。
5. 先问清楚行业需求，再生成配置和日报。

## 文件说明

- `01_系统说明.md`：给朋友看的系统解释。
- `02_Codex启动Prompt.md`：复制到 Codex 里即可启动配置采访。
- `03_初始化问卷.md`：需要先问清楚的问题清单。
- `04_日报模板.md`：标准日报结构。
- `05_信息源配置示例.md`：信息源和关键词配置样例。
- `06_运行流程.md`：每天如何跑、如何判断、如何归档。
- `SECURITY.md`：脱敏和上传边界。
- `scripts/sanitize-check.ps1`：上传前本地检查脚本。

## 快速使用

1. 先打开 `02_Codex启动Prompt.md`。
2. 把全文复制给自己的 Codex。
3. 按 Codex 的问题回答行业、方向、信息源和日报格式。
4. 让 Codex 生成你的第一版行业日报配置。
5. 连续试跑 3 天，再决定是否加脚本和定时任务。

## 不要直接上传的内容

- 个人 Obsidian vault
- 历史日报原文
- 日志文件
- API key、token、webhook
- 本地绝对路径
- 客户或朋友的真实资料
- 未脱敏的自动化脚本
